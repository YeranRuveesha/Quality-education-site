Splash Screen

Technical Description

The Splash Screen was created using HTML and CSS. Key features include:

  • HTML Meta Tag: Used the <meta http-equiv="refresh" content="4;url=home.html"> tag to automatically redirect users to the Home Page after 4 seconds.
  • CSS Animation: Created an animated loader using CSS keyframes to enhance user experience during the redirection.
  • Background Image: Used a full-screen background image with background-size: cover to ensure it scales properly across devices.
  • Overlay Effect: Added a semi-transparent overlay (rgba(0, 0, 0, 0.6)) to improve text readability over the background image.
  • Text Styling: Used custom fonts and colors (e.g., gold for the title) to align with the theme of quality education.

Accessibility

The Splash Screen includes:

  • Alternative text for the background image (implied through the use of a descriptive filename: education_background.jpg).
  • Proper heading structure (<h1> for the title and <p> for mission and group members).
  • High contrast between text and background for readability.

Content Page

Technical Description

The content page features a responsive layout with a fixed navigation bar, interactive sections, and a "Go to Top" button. It uses CSS for styling, including hover effects, two-column layout, and responsive images. The page ensures accessibility with alt text for images and easy navigation through internal links.

Accessibility

The page includes accessibility features like alt text for images, semantic HTML, focus styles for navigation links, and a responsive design for different screen sizes. It also ensures good color contrast for readability and easy keyboard navigation.

Challenges and Lessons Learned

One challenge was ensuring the background image on the Splash Screen scaled correctly on different screen sizes. I resolved this by using background-size: cover and testing on multiple devices. I learned the importance of responsive design and cross-browser compatibility.

Another challenge was making the scrollable section on the Volunteer Page responsive. I used CSS media queries to ensure it works well on mobile devices. I learned how to create dynamic layouts using Flexbox.

Compliance

All pages I implemented comply with Janet regulations for web accessibility. This includes:

  • Proper use of heading tags (<h1> to <h6>) for structure.
  • Alternative text for all images to ensure accessibility for screen readers.
  • Validation of HTML and CSS using the W3C Validator to ensure compliance with web standards.
