Feedback Page

Technical Description

The Feedback Page enables users to submit their thoughts on educational programs. It includes sections for personal details, feedback on educational content, and overall experience. The page features form validation, a star rating system, and dynamic character counters. It is styled with a modern design and is fully responsive for different screen sizes. Upon submission, a success message is displayed to confirm the feedback has been received.

Accessibility

The feedback page incorporates several accessibility features to enhance usability for all users. It uses semantic HTML elements like fieldset, legend, and label to improve screen reader compatibility. Required fields are clearly marked and announced by screen readers, while dynamic error messages help users navigate invalid or missing inputs. The page ensures clear focus styles for keyboard navigation, allowing users to interact seamlessly. Additionally, the form is fully navigable via keyboard and optimized for various screen sizes, ensuring a responsive experience across devices. These features contribute to an inclusive and accessible user experience.

Team Page

Technical Description

The "Team" page showcases the members of the Quality Education initiative with profile images, roles, and contributions. It uses a responsive grid layout with hover effects that reveal additional member details. Accessibility features include tabindex for keyboard navigation and descriptive alt text for images. The page is designed with smooth transitions and animations for an interactive user experience.

Accessibility

The "Team" page includes several accessibility features to ensure a better experience for all users. It supports keyboard navigation with the use of tabindex="0" on each team members profile, allowing users to easily navigate through the page. Descriptive alt text is provided for all profile images to assist screen readers for visually impaired users. Additionally, the page utilizes high-contrast text and background colors to enhance readability, making it accessible to users with visual impairments.

Content Page

Technical Description

The content page features a responsive layout with a fixed navigation bar, interactive sections, and a "Go to Top" button. It uses CSS for styling, including hover effects, two-column layout, and responsive images. The page ensures accessibility with alt text for images and easy navigation through internal links.

Accessibility

The page includes accessibility features like alt text for images, semantic HTML, focus styles for navigation links, and a responsive design for different screen sizes. It also ensures good color contrast for readability and easy keyboard navigation.

Challenges and Lessons Learned

Challenges included ensuring accessible navigation, responsive design, and image descriptions. The internal navigation was made keyboard-friendly, with focus styles for easy navigation. For responsiveness, media queries adjusted the layout for different screen sizes. Alt text was added to images for screen reader compatibility. Color contrast was improved for readability, and content was organized using semantic HTML for clarity. Lastly, the "Go to Top" button was made visible and functional with CSS and JavaScript for better user experience.

Compliance

The pages comply with Janet regulations by ensuring accessibility through features like alt text for images, semantic HTML for better screen reader support, and good color contrast. The design is responsive, adapting to various screen sizes. Additionally, clear navigation and a "Go to Top" button enhance usability, ensuring a user-friendly experience across devices.
